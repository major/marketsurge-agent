# marketsurge-agent: Initial Plan

Go CLI tool for AI agents to query the MarketSurge stock research API. Single binary, JSON-first output, auto-generated skill files for agent integration.

Inspired by [Google Workspace CLI (gws)](https://github.com/googleworkspace/cli) - single tool with skill files that teach agents how to use each command.

## Background

- **Python client library** (`~/git/major/marketsurge-python`): Async Python client for MarketSurge's GraphQL API. 20+ endpoints, 70+ data models, cookie-based auth, optional caching.
- **MCP server** (`~/git/major/marketsurge-mcp`): FastMCP server wrapping 9 curated tools for AI agents.
- **This project**: Native Go reimplementation of the GraphQL client as a CLI tool. Not a Python wrapper - standalone single binary.

## Tech Stack

| Choice | What | Why |
|--------|------|-----|
| Go 1.26 | Latest stable (April 2026) | Green Tea GC, `errors.AsType[T]()`, iterator-based reflection |
| urfave/cli v3 | CLI framework | Lighter than Cobra (468KB vs 4.3MB binary), simpler for subprocess pattern |
| `net/http` + `encoding/json` | GraphQL client | No external GraphQL lib needed for simple query execution |
| `modernc.org/sqlite` | Cookie extraction | Pure Go SQLite (no CGO) for reading browser cookie DBs |
| `log/slog` | Structured logging | Built-in, no external dep |
| `//go:embed` | Query embedding | Compile .graphql files into the binary |
| `stretchr/testify` | Test assertions | Standard Go testing practice |

## API Scope (Phase 1)

Starting with the 9 tools already proven useful in the MCP server, not the full 20+ endpoint surface.

### Commands

```text
Stock Research
  stock get <symbol>                        Single stock data (ratings, pricing, financials)
  stock analyze <symbol> [symbol...]        Multi-symbol analysis, partial failure tolerant
  fundamental get <symbol>                  Earnings/sales reported + estimates
  ownership get <symbol>                    Institutional ownership data
  rs-history get <symbol>                   RS rating history over time

Charts
  chart history <symbol>                    OHLCV price history
    --period daily|weekly                   Time period granularity
  chart markups <symbol>                    Chart annotations/markups

Catalog (Curated Lists)
  catalog list                              List available catalog entries (IBD 50, etc.)
  catalog run <name>                        Run a catalog entry, return matching stocks
    --limit N                               Max results (default: all)
    --offset N                              Pagination offset
    --fields f1,f2,...                       Select specific output fields
    --min-<field> N                          Numeric threshold filters
    --exclude-type <type>                   Exclude instrument subtypes (e.g., ETF)
```

### Global Flags

```text
  --jwt <token>           JWT auth token (highest priority)
  --format json|table|csv Output format (default: json)
  --fields f1,f2,...      Select specific output fields (recursive)
  --verbose               Enable debug logging to stderr
  --help                  Show help text
```

## Authentication Chain

Priority order (first match wins):

1. `--jwt` flag
2. `MARKETSURGE_JWT` environment variable
3. Browser cookie extraction (Firefox, then Chrome) + JWT exchange

Cookie extraction reads the MarketSurge session cookie from the browser's SQLite database, then exchanges it for a JWT against MarketSurge's token endpoint. Same flow as the Python library's `resolve_jwt` chain but reimplemented in Go.

## Output Contract

### Success

All data goes to stdout as JSON (default):

```json
{
  "data": { ... },
  "metadata": {
    "symbol": "AAPL",
    "timestamp": "2026-04-21T12:00:00Z"
  }
}
```

Multi-result commands (catalog run, stock analyze) return a JSON array wrapper:

```json
{
  "data": [ ... ],
  "metadata": {
    "total": 50,
    "limit": 25,
    "offset": 0,
    "timestamp": "2026-04-21T12:00:00Z"
  }
}
```

### Errors

Errors are JSON to stdout (so agents can parse them), with human-readable messages to stderr:

```json
{
  "error": {
    "code": "AUTH_FAILED",
    "message": "JWT expired",
    "hint": "Set MARKETSURGE_JWT or run with --jwt <token>"
  }
}
```

### Exit Codes

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | General error |
| 2 | Not found (symbol, catalog entry) |
| 3 | Authentication failure |
| 4 | API error (MarketSurge returned an error) |

## Skill Files

Auto-generated markdown files that teach AI agents how to use each command group. Generated by `marketsurge-agent skills generate`.

```text
skills/
  marketsurge-stock.md
  marketsurge-fundamental.md
  marketsurge-ownership.md
  marketsurge-rs-history.md
  marketsurge-chart.md
  marketsurge-catalog.md
```

Each skill file contains:

- Command syntax and flags
- Parameter descriptions and defaults
- Output schema with example JSON
- Error codes and recovery hints
- Common usage patterns for agents
- Workflow suggestions (e.g., "use catalog list first, then catalog run")

Agents load the relevant skill before calling the CLI, so they know exactly what commands exist, what arguments to pass, and how to interpret the output.

    main.go                     # Entrypoint, app setup, run
  internal/
    commands/

  go.sum

## Open Questions

- **Binary distribution**: goreleaser for cross-platform builds? Homebrew tap?
- **Cookie encryption**: Chrome encrypts cookies on Linux (DPAPI on Windows, Keychain on macOS) - need to handle decryption or fall back to Firefox-only initially
- **Rate limiting**: Does MarketSurge throttle? If so, add backoff/retry logic
- **Caching**: Should the CLI cache responses locally (like the Python lib's aiocache), or leave that to the calling agent?
- **Config file**: Support `~/.config/marketsurge-agent/config.yaml` for persistent settings (default format, JWT storage)?
